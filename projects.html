Transaction Simulation Model for Construcshare

Completed under DSA3101: Data Science in Practice

    This project is in collaboration with Construcshare, a company that provides a dealing platform for construction industry.
  • Problem statement: Understanding how each of the Construcshare website features affect site-engagement in terms of user interactions.
  • Simulation Model: Agent Based Model
  • Medium of Model: Mesa Python
  • Audience: Chief Data Officer and the development team of Construcshare
  • Model Purpose: Understand how the relationship between a feature on Construcshare and the users affects member website engagement and interaction
  • Model Parameters: Website Activity and Interaction when altering Platform Fee
  • Tools Used: Python, Docker, Git, API
  • Libraries Used: Mesa, NumPy, Math, Matplotlib, UUID, Pandas, Copy, Collections, Itertools, Numbers, Typing, Warnings, Networkx, Random, Datetime
  • Impact of solution for User: Construcshare can use these graphs as an indicator for their future predictions and strategies to gauge the customer behaviour so they can tailor features for each user.
